# Emilien : un agent PM/PO qui ne code pas mais manage des équipes d'agents — l'orchestration comme « management autonome »

**Résumé** — Powlisher raconte avoir construit avec Robin un agent PM/PO autonome, Emilien, qui ne code pas : il comprend un projet, rédige des briefs, planifie des roadmaps, fait apparaître des équipes d'agents via Orchestria, review leur travail, lance des audits parallèles, compare des branches concurrentes et retient les meilleurs outputs — jusqu'à mandater une seconde équipe IA pour vérifier la première quand il doute. Branché sur une mémoire persistante partagée, il devient meilleur expert métier et meilleur manager à mesure qu'il travaille (il pilote notamment des panes Superset, l'outil qui fait tourner 10+ agents en parallèle). L'idée forte : le goulot n'est plus le code mais l'orchestration, et elle aussi se délègue.
